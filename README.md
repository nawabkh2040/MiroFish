## ⚡ Project Overview

**MiroFish** is a next-generation AI prediction engine based on multi-agent technology. By extracting seed information from the real world (such as breaking news, policy drafts, financial signals), it automatically builds a high-fidelity parallel digital world. Within this space, thousands of intelligent agents with independent personalities, long-term memory, and behavior logic freely interact and evolve socially. You can dynamically inject variables through a "god's eye" perspective to accurately predict future trajectories——**allowing the future to be rehearsed in a digital sandbox, helping decisions win after hundred-fold simulations**.

> All you need to do: Upload seed materials (data analysis reports or interesting fiction stories) and describe your prediction needs in natural language</br>
> MiroFish will return: a detailed prediction report and a highly interactive digital world for deep exploration

### Our Vision

MiroFish is committed to building a swarm intelligence mirror that maps reality, breaking through the limitations of traditional prediction by capturing group emergence triggered by individual interactions:

- **Macro Level**: We are a simulation laboratory for decision-makers, enabling policies and public relations to experiment risk-free
- **Micro Level**: We are a creative sandbox for individual users, making it fun, engaging, and accessible to explore any "what if"

From serious prediction to engaging simulation, we make every scenario visible, making prediction of anything possible.

## 🔄 Workflow

1. **Graph Construction**: Reality seed extraction & Individual and collective memory injection & GraphRAG construction
2. **Environment Setup**: Entity relationship extraction & Character generation & Environment configuration with simulation parameters
3. **Begin Simulation**: Dual-platform parallel simulation & Automatic prediction requirement parsing & Dynamic memory updates
4. **Report Generation**: ReportAgent equipped with rich toolset for deep interaction with the simulated environment
5. **Deep Interaction**: Dialogue with any character in the simulated world & Dialogue with ReportAgent

## 🚀 Quick Start

### I. Source Code Deployment (Recommended)

#### Prerequisites

| Tool | Version | Description | Check |
|------|---------|-------------|-------|
| **Node.js** | 18+ | Frontend runtime environment, includes npm | `node -v` |
| **Python** | ≥3.11, ≤3.12 | Backend runtime environment | `python --version` |
| **uv** | Latest | Python package manager | `uv --version` |

#### 1. Configure Environment Variables

```bash
# Copy example configuration file
cp .env.example .env

# Edit .env file and add necessary API keys
```

**Required Environment Variables:**

```env
# LLM API configuration (supports any LLM API in OpenAI SDK format)
# Recommended: Use Alibaba Bailian qwen-plus model: https://bailian.console.aliyun.com/
# Note: High consumption, try simulations with less than 40 rounds first
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Zep Cloud Configuration
# Free monthly quota supports basic usage: https://app.getzep.com/
ZEP_API_KEY=your_zep_api_key
```

#### 2. Install Dependencies

```bash
# One-command installation of all dependencies (root + frontend + backend)
npm run setup:all
```

Or step-by-step installation:

```bash
# Install Node dependencies (root + frontend)
npm run setup

# Install Python dependencies (backend, automatically creates virtual environment)
npm run setup:backend
```

#### 3. Start Services

```bash
# Start both frontend and backend simultaneously (run in project root)
npm run dev
```

**Service URLs:**
- Frontend: `http://localhost:3000`
- Backend API: `http://localhost:5001`

**Start Services Individually:**

```bash
npm run backend   # Start backend only
npm run frontend  # Start frontend only
```

### II. Docker Deployment

```bash
# 1. Configure environment variables (same as source code deployment)
cp .env.example .env

# 2. Pull images and start
docker compose up -d
```

Default reads `.env` from root directory and maps ports `3000 (frontend) / 5001 (backend)`

> Pre-configured mirror URLs are provided in comments in `docker-compose.yml`, replace as needed
